# Methodology

## Theoretical Foundations

Semantic Equity Research integrates three complementary analytical frameworks:

1. **Traditional Equity Analysis** - Fundamental, technical, and sentiment analysis
2. **Semantic Knowledge Graphs** - Network-based representation of relationships
3. **Cognitive Science** - Bias detection and thinking pattern analysis

---

## 1. Traditional Equity Analysis

### Fundamental Analysis
- Revenue and earnings trends
- Margin analysis and profitability
- Balance sheet health
- Peer comparison and relative valuation

### Technical Analysis
- Support and resistance levels
- Momentum indicators (RSI, MACD)
- Volume analysis
- Chart patterns

### Sentiment Analysis
- Analyst ratings and price targets
- Insider trading signals
- Institutional ownership changes
- Short interest dynamics

## 2. Semantic Knowledge Graphs

### Why Knowledge Graphs?

Traditional analysis presents information linearly:
> "The company has strong growth because of AI demand."

Knowledge graphs reveal **network structures**:
```
[[AI Demand]] → enables → [[Revenue Growth]]
[[Revenue Growth]] → supports → [[R&D Investment]]
[[R&D Investment]] → strengthens → [[AI Capabilities]]
[[AI Capabilities]] → drives → [[AI Demand]]
```

This reveals a **reinforcing feedback loop** that linear analysis might miss.

### Ontology Principles

Based on the InfraNodus Ontology Creator methodology:

1. **Network Structure Over Hierarchy**
   - Avoid hub-and-spoke patterns
   - Create distributed connections
   - Target 2-4 connections per node

2. **Relationship Typing**
   - Causal: causes, enables, threatens
   - Structural: partOf, dependentOn
   - Temporal: precedes, follows
   - Evaluative: supports, opposes

3. **Balance Requirements**
   - Bullish and bearish pathways
   - Macro and micro factors
   - Short and long-term considerations

### [[Wikilinks]] Syntax

The `[[entity]]` syntax:
- Marks semantic entities explicitly
- Enables automated graph construction
- Compatible with InfraNodus and Obsidian
- Creates machine-readable output

## 3. Cognitive Framework

### Cognitive Variability Model

Derived from InfraNodus research on thinking patterns:

| State | Graph Topology | Thinking Pattern |
|-------|---------------|------------------|
| BIASED | Star (one hub) | Tunnel vision |
| FOCUSED | Dense cluster | Coherent but narrow |
| DIVERSIFIED | Multiple clusters | Healthy multi-view |
| DISPERSED | Scattered nodes | Unfocused, chaotic |

### State Detection Method

1. **Narrative Analysis**: Identify dominant story in market
2. **Bias Checklist**: Screen for common cognitive biases
3. **Graph Topology**: Analyze Event Ontology structure
4. **Classify State**: Apply classification formula

### Critical Thinking Integration

Based on structured analytic techniques:

**Assumption Surfacing**
- Make implicit beliefs explicit
- Rate confidence levels
- Assess impact if wrong

**Devil's Advocate**
- Generate strongest counter-argument
- Find historical parallels of failure
- Propose alternative interpretations

**Pre-Mortem Analysis**
- Imagine future failure
- Work backwards to causes
- Identify early warning signs

## Validation

### Quality Indicators

**Event Ontology**:
- ✓ 15+ relationship statements
- ✓ No single entity >30% of statements
- ✓ Both bullish and bearish pathways
- ✓ Macro-micro connections present

**Critical Assumptions**:
- ✓ 5+ assumptions surfaced
- ✓ Confidence levels assigned
- ✓ Impact assessments specific
- ✓ Devil's Advocate genuinely challenging

**Cognitive State**:
- ✓ State clearly classified
- ✓ Reasoning provided
- ✓ Biases explicitly checked
- ✓ Recommendation actionable
